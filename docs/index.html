Pitfall: P22: Using different naming conventions in the ontology.

Explanation: Due to the re-use of concepts from other ontologies, some of the classes and properties in the AFMO use different naming conventions. Original AFMO classes use TitleCase for classes (e.g. MonitoredPerson) and camelCase for object and data properties (e.g. hasAgeGroup). Other imported ontologies have different naming conventions, such as the use of underscores and numeric codes in the Disease Ontology (e.g. DOID_0060224 for "atrial fibrillation").
